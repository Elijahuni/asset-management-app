# AI 에이전트 프롬프트 및 작업 이력 (Prompt History)

이 문서는 개발 과정에서 AI 에이전트에게 내린 요청(프롬프트)과 그에 따른 구현 결과, 그리고 문제 해결 이력을 기록하는 버전 관리 파일입니다. 
나중에 특정 기능이 어떻게 만들어졌는지, 어떤 의도로 코드가 작성되었는지 추적하기 위해 사용합니다.

---

## 템플릿 (작성 가이드)

### [날짜 또는 버전] - [구현 및 수정된 기능명]
* **요청자(User) 프롬프트:** 에이전트에게 요청한 정확한 내용
* **에이전트(Agent) 작업 내용:** 에이전트가 생성한 주요 로직, 생성/수정된 파일명
* **주요 결정 사항 및 의도:** 이 방식을 택한 이유 또는 해결한 에러
* **결과 및 참고 링크:** (선택 사항) 관련된 이슈 번호, 참고 링크 등

## 작업 이력 기록 (예시)

### 2026-04-20 - 자산 관리 대시보드 및 기초 설정 구현
* **요청자(User) 프롬프트:** "사내 자산 관리를 위한 Streamlit 기반 웹앱을 만들고, Supabase를 연동해줘. 메인에 대시보드를 추가하고 자산 통계를 보여줘."
* **에이전트(Agent) 작업 내용:** 
  * `app.py` 및 `pages/1_📊_대시보드.py` 생성
  * `schema.sql`을 통한 초기 데이터베이스 구조 설계
  * `utils/db.py`에 Supabase 연동 코드 작성
* **주요 결정 사항 및 의도:** 멀티페이지 앱을 위해 `pages/` 디렉토리 구조를 사용함. 보안을 위해 Supabase 연동 키는 `.streamlit/secrets.toml`에 분리 저장.
* **결과:** 로컬에서 `streamlit run app.py` 실행 성공, 초기 형태 완성.

### 2026-04-21 - 에이전트 구현 히스토리 문서화 파일 생성
* **요청자(User) 프롬프트:** "현재까지 애이전트로 구현한 코드에 대해 히스토리를 .md파일에 저장해줘"
* **에이전트(Agent) 작업 내용:** 
  * 로컬 프로젝트 구조 참조하여 `AGENT_HISTORY.md` 파일 생성하여 주요 내역 요약.
* **결과:** 완료

### 2026-04-21 - 자산 상태 기반 라이프사이클(Soft Delete) 구현
* **요청자(User) 프롬프트:** "next step 1번부터 순차적으로 고도화해주세요. 빨리 안해도 되지 분석을 먼저하고 기능이 완벽하게 될수 있도록 구성해야 합니다."
* **에이전트(Agent) 작업 내용:** 
  * `utils/db.py`에 물리적 삭제(Hard Delete)를 막고 상태를 '폐기'로 바꾸는 `retire_asset` 함수 구축
  * `pages/2_📋_자산목록.py` 하단에 관리자 전용 폐기 처리 폼 생성 및 기본 리스트에서 폐기 자산 숨김 처리
* **주요 결정 사항 및 의도:** 기업 자산 관리 원칙에 따라 데이터베이스 레코드를 보존하면서 확인 및 감사가 가능하도록 Soft Delete 방식으로 기능 구현. 보안을 위해 폐기 폼은 `is_admin` 세션 전용으로 제한함.
* **결과:** 구현 완료, v1.1.0 업데이트 

### 2026-04-21 - Role-Based Access Control (RBAC) 고도화
* **요청자(User) 프롬프트:** "step1번이 완료가 되었는지 확인후 step 2번을 순차적으로 고도화해주세요. 빨리 안해도 되지 분석을 먼저하고 기능이 완벽하게 될수 있도록 구성해야 합니다."
* **에이전트(Agent) 작업 내용:** 
  * `pages/1_📊_대시보드.py` 하단의 알림 섹션을 관리자 세션 검증 후 렌더링.
  * `pages/4_🤝_대여관리.py` 탭2에서 일반 직원일 경우 자신의 대여 자산만 필터링하여 노출하도록 변경 및 과거 반납 내역 뷰 추가.
* **주요 결정 사항 및 의도:** 보안과 프라이버시 보호를 위해 일반 임직원이 불필요한 전체 장비 유지보수 데이터나 타인의 대여 현황을 확인하지 못하도록 UI 레벨에서 강력한 접근 권한 제어를 적용함.
* **결과:** 구현 완료, v1.2.0 배포

### 2026-04-21 - 자산 감가상각(Depreciation) 자동 계산기 탑재
* **요청자(User) 프롬프트:** "바코드 도입은 지금하지 말고 감가상각 계산기 탑재 먼저해줘"
* **에이전트(Agent) 작업 내용:** 
  * `utils/depreciation.py` 파일 생성하여 정액법(일수 단위 비례 차감) 기반의 잔존가치 산출 로직 구축.
  * 기존 데이터프레임 `DataFrame.apply()`를 활용하여 `1_대시보드`에는 총합을 메트릭 카드로 구성하고, `2_자산목록`에는 개별 장비 가치를 조회할 수 있도록 변경.
* **주요 결정 사항 및 의도:** 기업 회계의 표준 방식 중 하나인 내용연수 5년(60개월) 정액법을 디폴트 상수로 주입하여 초기 구축 복잡도를 낮춤. 임직원 간 자산 가치 유출 방지를 위해 회계 정보 접근을 `is_admin` 모드로 철저히 격리 처리.
* **결과:** 구현 및 V1.3.0 배포 완료
