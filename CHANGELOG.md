# Changelog

이 파일은 프로젝트의 버전별 변경 사항을 요약하여 기록합니다.

## [v1.3.0] - 2026-04-21

### 추가됨 (Added)
- `utils/depreciation.py`: 자산 가치 하락 점검을 위한 5년 선형 정액법 감가상각 (Depreciation) 자동 계산 로직.
- `pages/1_📊_대시보드.py`: 관리자 모드 전용 '회사 회계 자산 요약' 카드 도입 (총 구매 원가 합계, 총 추정 잔존 가치, 가치 보존율 표시 기능).
- `pages/2_📋_자산목록.py`: 관리자 권한 조회 시 목록 데이터에 '현재가치(추정)' 열 자동 추가.

## [v1.2.0] - 2026-04-21

### 추가됨 (Added)
- `pages/4_🤝_대여관리.py`: 일반 임직원을 위한 '나의 과거 반납 이력' 뷰 섹션 추가.

### 변경됨 (Changed)
- `pages/1_📊_대시보드.py`: 자산(보증/유지보수) 경고 알림 섹션을 관리자 전용으로 감춤 처리.
- `pages/4_🤝_대여관리.py`: 일반 임직원은 본인이 대여한 내역만 확인하고 반납할 수 있도록 필터링 로직 도입.

## [v1.1.0] - 2026-04-21

### 추가됨 (Added)
- `utils/db.py`: 자산 '폐기(Soft Delete)' 처리를 위한 `retire_asset` 백엔드 함수 및 이력 로깅 추가.
- `pages/2_📋_자산목록.py`: 관리자(Admin) 전용 자산 상태 관리 UI(폐기 폼) 추가.
- `pages/2_📋_자산목록.py`: 폐기된 자산을 기본 목록에서 숨기고 볼 수 있는 '폐기 자산 포함보기' 필터 체크박스 추가.

## [v1.0.1] - 2026-04-21

### 추가됨 (Added)
- `AGENT_HISTORY.md`: 에이전트를 통해 완성된 현재까지의 앱 구현 히스토리 문서 작성.
- `PROMPT_HISTORY.md`: 개발 과정에서 에이전트와 나눈 요청(프롬프트) 및 작업 이력을 기록하는 템플릿 파일 추가.
- `CHANGELOG.md`: 프로젝트의 버전을 업데이트하고 파일 변경 이력을 요약 관리하기 위한 문서 생성.

### 변경됨 (Changed)
- 기존 자산 관리 애플리케이션의 구현 내역 정리 및 문서화 구조 확립.

## [v1.0.0] - 2026-04-20

### 추가됨 (Added)
- `app.py`: Streamlit 앱 메인 진입점 및 로그인/인증 로직 구현.
- `pages/*`: 멀티페이지 라우팅 구현 (대시보드, 자산목록, 자산등록, 대여관리, 이력조회).
- `schema.sql`: Supabase 연동을 위한 users, assets, asset_history 테이블 구조 초기 셋업.
- `utils/db.py`: 데이터베이스 CRUD 처리를 위한 유틸리티 함수 로직 추가.
- `GUIDE.md`: 로컬 실행 환경 및 Streamlit Cloud 배포 가이드 문서 파일 작성.
