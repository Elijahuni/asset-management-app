# 사내 자산 관리 애플리케이션 구현 히스토리

본 문서는 현재까지 AI 에이전트와 함께 구현한 사내 자산 관리 애플리케이션(Streamlit 기반)의 개발 내역 및 구조를 요약한 히스토리 파일입니다.

## 1. 프로젝트 주요 목표 (완료됨)
- **목적:** Python, Streamlit, Supabase를 활용한 안전한 다중 페이지 사내 자산 관리 웹 애플리케이션 구축
- **주요 기능:**
  - 자산 등록, 조회, 수정, 삭제 (CRUD)
  - 자산 대여 및 반납 관리 (대여 추적)
  - 자산 변경 이력 및 대여 이력 자동 기록
  - 대시보드를 통한 자산 통계 및 유지보수 알림
  - 사용자 인증 및 권한 기반 접근 제어 (Supabase RLS 활용)
  - Streamlit Cloud 배포 가이드라인 작성

## 2. 구현된 파일 및 모듈 구조

### 메인 애플리케이션 및 설정
- **`app.py`**: 서비스의 메인 진입점. Streamlit 기본 설정 및 사용자 로그인/로그아웃 처리를 담당하는 인증 페이지 로직 포함.
- **`.streamlit/secrets.toml`**: Supabase 연동에 필요한 `SUPABASE_URL` 및 `SUPABASE_KEY` 등의 환경 변수 및 보안 자격 증명 파일.
- **`requirements.txt`**: `streamlit`, `supabase`, `pandas`, `plotly` 등 프로젝트 실행에 필요한 패키지 의존성 목록.
- **`GUIDE.md`**: 로컬 환경 설정 및 Streamlit Cloud로의 배포 가이드 문서.

### 데이터베이스 및 유틸리티
- **`schema.sql`**: Supabase에 적용된 데이터베이스 스키마 정의.
  - `users`: 사용자 정보 테이블
  - `assets`: 자산 정보 테이블 (상태, 유지보수 일정 등 포함)
  - `asset_history`: 자산 상태 변경 및 대여/반납 내역을 추적하는 이력 테이블
- **`utils/db.py`**: Supabase 클라이언트 초기화 및 데이터베이스와 통신하기 위한 주요 함수들(사용자 인증 검증, 자산 및 이력 정보 CRUD 등) 구현.

### 자산 관리 기능 페이지 (Pages)
- **`pages/1_📊_대시보드.py`**: 전체 자산 요약 통계(총 자산 수, 대여 중인 자산 수 등) 및 유지보수가 임박하거나 필요한 자산 목록 알림 대시보드.
- **`pages/2_📋_자산목록.py`**: 전체 자산 목록 조회 및 필터링 기능. 개별 자산의 상세 정보 확인, 정보 수정, 그리고 자산 삭제 기능 제공.
- **`pages/3_➕_자산등록.py`**: 새로운 자산의 정보(모델명, 카테고리, 구매일, 유지보수 주기 등)를 입력하고 데이터베이스에 등록하는 폼.
- **`pages/4_🤝_대여관리.py`**: 보유한 자산에 대해 사용자에게 대여 및 반납 처리를 수행하는 관리 페이지.
- **`pages/5_🕒_이력조회.py`**: 특정 자산의 대여/반납 기록, 상태 변경 등 세부적인 변동 이력을 시간순으로 조회하는 기능.

## 3. 핵심 기술 및 라이브러리
- **Frontend / UI:** Streamlit (멀티 페이지 구성, 데이터 시각화)
- **Backend / Database:** Supabase (PostgreSQL 원격 데이터베이스, Authentication, RLS 보안 정책 지원)
- **Data Processing:** Pandas (데이터프레임 처리)
- **Language:** Python
